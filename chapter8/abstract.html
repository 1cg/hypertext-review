Abstract

A number of navigational tools exist for hypertext systems. Authoring guidelines have also been proposed for the organization of information in hypertext systems. However, there has been no systematic and comprehensive approach towards the design of user interfaces for hypertext systems. This paper is an attempt to apply a set of user interface design guidelines to a hypertext framework based on a cognitive model. This framework (see also other models and framworks had classified nodes and links into various semantic types. We believe that such a classification is of great importance in developing an appropriate design metaphor/user interface for a hypertext system. A systematic approach to user interface design would also reduce functional opacity and system opacity.

Keywords: Hypertext, User Interface Design, Cognitive Models, Guidelines.
